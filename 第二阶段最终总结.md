### 1. 检索结果二次权限校验 ⭐⭐⭐

**文件：** `rag-wiki-python/app/services/post_search_validator.py` (353行)

**核心能力：**
- ✅ 六层权限验证（租户/安全等级/空间/部门/角色/文档级）
- ✅ 结果脱敏处理（根据用户等级自动脱敏）
- ✅ 违规访问统计与日志
- ✅ 双重保险机制（前置过滤 + 后置校验）

**安全架构：**
```
用户请求
  ↓
[第一层] 前置权限验证 → 快速拒绝非法请求
  ↓
[第二层] 带权限过滤的向量检索
  ↓
[第三层] 二次权限校验 → 再次验证检索结果
  ↓
[第四层] 结果脱敏 → 根据等级脱敏敏感内容
  ↓
返回结果
```

**使用示例：**
```python
from app.services.post_search_validator import validate_search_results

filtered_results = validate_search_results(
    results=search_results,
    user_id="user_001",
    tenant_id="tenant_001",
    dept_id="dept_001",
    role_ids=["role_admin"],
    security_level=4,
    allowed_space_ids=["space_001"],
)
```

**集成点：**
- 已集成到 RAG 引擎 [rag_engine.py](file:///Users/xiong/Desktop/openclaw/project/rag-wiki/rag-wiki-python/app/services/rag_engine.py#L408-L436)
- API层自动调用，无需手动处理

### 4. 文档解析质量分层保障 ⭐⭐

**文件：** `rag-wiki-python/app/services/quality_assurance.py` (373行)

**核心能力：**
- ✅ 三级质量保障（基础→增强→人工复核）
- ✅ 自动质量评估（置信度/提取率/错误率）
- ✅ 智能分级（HIGH/MEDIUM/LOW/FAILED）
- ✅ 人工复核队列管理
- ✅ 质量指标追踪

**质量评估维度：**
| 维度 | 权重 | 说明 |
|------|------|------|
| 基础置信度 | 40% | 解析器自身评分 |
| 文本提取率 | 30% | 成功提取的文本比例 |
| 错误率 | 20% | 解析错误占比 |
| 内容完整性 | 10% | 表格/图片处理完整性 |

**质量分级标准：**
- **HIGH** (≥0.85): 基础解析，直接使用
- **MEDIUM** (≥0.65): 需要增强解析（OCR/版面分析）
- **LOW** (≥0.40): 需要人工复核
- **FAILED** (<0.40): 解析失败，必须人工处理

**使用示例：**
```python
from app.services.quality_assurance import assess_document_quality

result = assess_document_quality(
    document_id="doc_001",
    raw_result={"chunks": [...]},
    metrics={
        "text_extraction_rate": 0.85,
        "table_count": 3,
        "image_count": 5,
        "page_count": 50,
        "error_count": 2,
    },
)

print(f"质量等级: {result.quality_level.value}")
print(f"当前阶段: {result.current_stage.value}")
print(f"需要复核: {result.needs_review}")
```
